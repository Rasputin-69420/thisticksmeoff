## Data Dump Mode (Current Operating Mode)

You will keep sending raw source lists (big JSON objects with "data_sources").

**My job:**
- Save every dump you send into `data-pipeline/raw/dumps/` with a timestamp.
- Run the new intake tool (`ingest_source_dump.py`) to automatically filter redundancies (URL + fuzzy title matching against the two master source files).
- Only integrate the genuinely new items into:
  - `data/lyme_ags_database/lyme_ags_sources.json`
  - `data/additional_sources.json`
- Log every batch in `DATA_CONTRIBUTIONS.md` (Incoming Data Log table).
- Keep both `website-testnet/` and `my-website/` in sync.

The goal is a clean, non-duplicative data set while you freely dump sources. The tool is now the official gatekeeper.
